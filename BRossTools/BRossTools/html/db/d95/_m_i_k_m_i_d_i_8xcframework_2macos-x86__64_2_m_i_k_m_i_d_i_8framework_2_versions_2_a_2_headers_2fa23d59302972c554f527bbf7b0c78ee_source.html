20 /**
21  * MIKMIDIMappingManager provides a centralized way to manage an application's
22  * MIDI mappings. It handles both bundled (built in) mapping files, as well
23  * as user-customized mappings. It will automatically load both bundled and
24  * user mappings from disk. It will also save user mappings to an appropriate
25  * location on disk, providing an easy to way to support the ability to import user
26  * mappings to an applicaation.
27  *
28  * MIKMIDIMappingManager is a singleton.
29  */
30 @interface MIKMIDIMappingManager : NSObject

80 /**
81  * Import and load a user-supplied MIDI mapping XML file. This method loads the MIDI mapping
82  * file specified by URL and adds it to the set returned by -userMappings. The newly imported mapping
83  * file is also copied into the application's user mapping folder so it will be loaded again automatically
84  * on subsequent launches.
85  *
86  * If shouldOverwrite is YES, and an existing file with the same URL as would be used
87  * to save the imported mapping already exists, the existing file is deleted. Otherwise,
88  * a unique name is used for the newly imported mapping, preserving both mapping files.
89  *
90  * @param URL The fileURL for the mapping file to be imported. Should not be nil.
91  * @param shouldOverwrite YES if an existing mapping with the same file name should be overwitten, NO to use a unique file name for the newly imported mapping.
92  * @param error Pointer to an NSError used to return information about an error, if any.
93  *
94  * @return An MIKMIDIMapping instance for the imported file, or nil if there was an error.
95  */
96 - (nullable MIKMIDIMapping *)importMappingFromFileAtURL:(NSURL *)URL overwritingExistingMapping:(BOOL)shouldOverwrite error:(NSError **)error;
