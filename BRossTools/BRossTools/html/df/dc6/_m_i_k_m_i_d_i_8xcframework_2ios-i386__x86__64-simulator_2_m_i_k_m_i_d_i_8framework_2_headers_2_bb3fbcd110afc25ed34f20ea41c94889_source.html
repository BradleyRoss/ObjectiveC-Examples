37 /**
38  * Convenience method for creating a single, 14-bit control change command from its component
39  * messages. The two commands passed into this method must comply with the MIDI specification
40  * for 14-bit control change messages.
41  *
42  * The MIDI spec allows for 14-bit control change commands. These are actually sent as two
43  * sequential commands where the second command has a controller number equal to the first
44  * message's controllerNumber plus 32, and whose value is the least significant 7-bits of
45  * the 14-bit value.
46  *
47  * @note This method is used internally by MIKMIDI, to coalesce incoming 14-bit control change commands.
48  * it is not generally useful to external users of MIKMIDI. If you're simply trying to create a new
49  * MIKMIDIControlChangeCommand instance, you should use plain alloc/init instead.
50  *
51  * @param msbCommand The command containing the most significant 7 bits of value data (ie. the first command).
52  * @param lsbCommand The command containing the least significant 7 bits of value data (ie. the second command).
53  *
54  * @return A new, single MIKMIDIControlChangeCommand instance containing 14-bit value data, and whose
55  * fourteenBitCommand property is set to YES.
56  */
57 + (nullable instancetype)commandByCoalescingMSBCommand:(MIKMIDIControlChangeCommand *)msbCommand andLSBCommand:(MIKMIDIControlChangeCommand *)lsbCommand;
