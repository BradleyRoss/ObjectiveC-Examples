13 /**
14  * A MIDI System Keep Alive message (aka Active Sensing).
15  * Keep alive messages are sent repeatedly
16  * to tell the receiver of them that the MIDI connection is alive and working.
17  * Not all devices will send these at all.
18  *
19  * Per the MIDI spec, if a device *does* receive a keep alive message, it should
20  * expect to receive another no more than 300 ms later. If it does not, it can
21  * assume the connection has been terminated, and turn off all currently active
22  * voices/notes.
23  *
24  * Note that MIKMIDI doesn't (currently) implement this behavior at all, and it
25  * is up to MIKMIDI clients to implement it if so desired.
26  */
