14 /**
15  * Subtypes of MIKMIDIMetaEvent. You should use the corresponding meta subtypes in MIKMIDIEventType when
16  * initializing an event with -initWithTimeStamp:midiEventType:data: or similar methods.
17  *
18  * The reason for a separate enum here, even though there is a 1 to 1 correspondence with values in
19  * MIKMIDIEventType is that these values are dictated by the MIDI standard, and overlap values defined
20  * for MusicEventType. Having these separately defined allows us to effectively "flatten" MIKMIDIEventType
21  * to treat meta event subtypes as first class event types.
22  */
