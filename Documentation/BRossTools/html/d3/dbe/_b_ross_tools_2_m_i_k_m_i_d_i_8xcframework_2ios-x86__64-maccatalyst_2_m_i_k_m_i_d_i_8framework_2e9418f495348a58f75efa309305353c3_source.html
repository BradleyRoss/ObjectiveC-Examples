20 /**
21  * MIKMIDIConnectionManager can be used to manage a set of connected devices. It can be configured to automatically
22  * connect to devices as they are added, and disconnect from them as they are removed. It also supports saving
23  * the list of connected to NSUserDefaults and restoring them upon relaunch.
24  *
25  * The use of MIKMIDIConnectionManager is optional. It is meant to be useful in implementing functionality that
26  * many MIDI device enabled apps need. However, simple connection to devices or MIDI endpoints can be done with
27  * MIKMIDIDeviceManager directly, if desired.
28  */
29 @interface MIKMIDIConnectionManager : NSObject

135 /**
136  * Controls whether the receiver's availableDevices property includes virtual devices (i.e. devices made
137  * up of automatically paired virtual sources and destinations).
138  *
139  * If this property is YES (the default), the connection manager will attempt to automtically related
140  * associated virtual sources and destinations and create "virtual" MIKMIDIDevice instances for them.
141  *
142  * If this property is NO, the connection manager's availableDevices array will _only_ contain non-virtual
143  * MIKMIDIDevices.
144  *
145  * For most applications, this should be left at the default YES, as even many physical MIDI devices present as
146  * "virtual" devices in software.
147  *
148  * @note: The caveat here is that this relies on some heuristics to match up source endpoints with destination endpoints.
149  * These heuristics are based on the way certain common MIDI devices behave, but may not be universal, and therefore
150  * may miss, or fail to properly associate endpoints for some devices. If this is a problem for your application,
151  * you should obtain and connect to virtual sources/endpoints using MIKMIDIDeviceManager directly instead.
152  */
153 @property (nonatomic) BOOL includesVirtualDevices; // Default is YES

221 /**
222  * A connection manager's delegate can implement this method to be notified when a connected device is disconnected,
223  * either because -disconnectFromDevice: was called, or because the device was unplugged.
224  *
225  * If a MIDI device is disconnected between sending a note on message and sending the corresponding note off command,
226  * this can cause a "stuck note" because the note off command will now never be delivered. e.g. a MIDI piano keyboard
227  * that is disconnected with a key held down. This method includes an array of these unterminated note on commands (if any)
228  * so that the receiver can appropriately deal with this situation. For example, corresponding note off commands could
229  * be generated and sent through whatever processing chain is processing incoming MIDI commands to terminate stuck notes.
230  *
231  * @param manager An instance of MIKMIDIConnectionManager.
232  * @param device The MIKMIDIDevice that was disconnected.
233  * @param commands An array of note on messages for which corresponding note off messages have not yet been received.
234  */
235 - (void)connectionManager:(MIKMIDIConnectionManager *)manager deviceWasDisconnected:(MIKMIDIDevice *)device withUnterminatedNoteOnCommands:(MIKArrayOf(MIKMIDINoteOnCommand *) *)commands;
